Parser assignment

Background

You are given a particially completed parser, which is intended to check whether an input program is grammatically correct.  Some specific features:
  • As in the Scanner project, the parser ignores (but copies) "meta-statements".
  • The LL(1) grammar definition of the language of interest is here.
  • For each input program, the parser reports "pass" or "error" as the result of grammar analysis.  It suffices to include only one of these two words in the output (stdout or stderr) of the program.
  • The parser counts the number of global and local variables (through <data decls>), functions (through <func list> but not function declarations), and statements (through <statements>) in correct programs.  It outputs the result in the following format: variable _num_variables  function _num_functions statement _num_statements.  For example, for a correct program with 4 variables, 2 functions and 6 statements, the compiler should output at least the following:
    • Pass variable 4 function 2 statement 6

What You Need To Do

For this project, you are required to fill in the missing functions in the parser.cpp file.  These functions have a TODO comment so you can easily find them.  You will need to read through the code to learn how it works, and develop an understanding of how the parser works.  You do not need to create any additional functions or modify any other part of the program for it to work.  You are graded on your implementations of the missing functions.  If they are implemented correctly, the program will work as described in the Background section.

What to submit

  • Submit your solution in a single compressed file (e.g., .zip/.tar/.rar/.gz). Include a ReadMe describing your implementation and the changes you made.
